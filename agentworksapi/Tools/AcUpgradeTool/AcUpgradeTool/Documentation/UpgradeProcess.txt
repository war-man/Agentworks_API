1. Run the AcUpgradeTool from command prompt with a single nameless parameter of the path to the AC WSDL being processed
Example:
"C:\AcUpgradeTool.exe" "D:\AgentConnect.wsdl"

IMPORTANT: "output" directory with all auto-generated files will be written to the current command prompt directory.

2. Remove the following files & directories from the AgentWorksAPI solution in visual studio
	- AwApi.EntityMapper
        * ModelExtensions directory
        * ViewModelExtensions directory
        * AwAgentConnectMapper.cs

	- AwApi.ViewModels
        * Transaction directory

	- MoneyGram.AgentConnect.DomainModel
        * Transaction directory

	- MoneyGram.AgentConnect.Repository\EntityMapper
        * DomainModelExtensions directory
        * ServiceModelExtensions directory
        * AcMapper.cs

3. The output is organized into directories by project name.
Within visual studio, add the folders from the AcUpgradeTool output directory to the proper project
(e.g. output\AwApi.ViewModels\Transaction should be copied to the AwApi.ViewModels project)
The files get copied, but you still need to right-click on the folder and Add Existing Item to add the files to the project

Directories that must be done manually (Visual Studio may not add these subdirectories)
MoneyGram.AgentConnect.DomainModel\Transaction\Request
MoneyGram.AgentConnect.DomainModel\Transaction\Response
AwApi.ViewModels\Transaction\Request
AwApi.ViewModels\Transaction\Response

4. Within WebStorm, remove the following directory:
	- app\js\viewModels\transaction

5. Within WebStorm, copy the JavaScript output directory to the matching directory in WebStorm
	- AwStaticContent -> AwStaticContent

6. If changes have been made to the autogenerated files, they will need to be reproduced manually.
As of the writing of this document, the following changes need to be made:
	
	- AwApi.ViewModels\Transaction\ProductProfileItemType & MoneyGram.AgentConnect.DomainModel\Transaction\ProductProfileItemType extend ProfileItemType
		
		public sealed class ProductProfileItemType : ProfileItemType

    - Remove "sealed" from AwApi.ViewModels\Transaction\ProfileItemType.cs and & MoneyGram.AgentConnect.DomainModel\Transaction\ProfileItemType.cs

7. Rebuild the Web API solution and ensure there are no build errors and run the integration test suite.
